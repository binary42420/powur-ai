Please note that it may take up to 10 seconds for the response to process after uploading a document. Additional options are available by clicking the three dots to the right.
If you do not have a bill on hand, select a pre-uploaded example from the "Sample" dropdown. Some samples were intentionally uploaded in a blurry format to demonstrate the processor's flexibility.
It should be noted that Spanish was not included in the machine learning model's training. However, one of the samples is printed in Spanish to showcase the processor's adaptability.
- The "Entities" fields are custom-trained for our use-case. The OCR processor is trained to extract these fields as separate entities, for easy input into a database.
- If the processor confidence is under a specified threshold, for an entity field, it will mark all the other fields, before forwarding the document to a specialist to complete the labeling manually. Once labeled, it is then re-submitted to the processor to retrain, and become smarter over time.
- The meter number was not identified on the SDG&E Sample bill. While it did successfully read out the data, the processor simply hasn't been trained with enough data to always identify the entities. With additional up-training of the model, the processor will improve over time.
The current training set comprises only a small number of sample documents for SDG&E. A total of only 40 between SCE, PG&E, and SDG&E Even with such a small amount of training data, you will find that it is still impressively good at finding these desired entities and labeling them.
The documents that don't meet a pre-defined threshold, will be forwarded for manual review. This is not a bad thing however, it simply means there's less of a chance that erroneous data gets passed into the main database. Once reviewed, by re-processing the labeled data into the processor, thats the very best way to improve the processor over time.
